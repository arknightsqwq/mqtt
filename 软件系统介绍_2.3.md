# 2.3 软件系统介绍

## 2.3.1 软件整体介绍

### 一、系统概述

本系统是一套完整的 **IoT 智能头盔远程监测平台**，采用 B/S 架构（Browser/Server），由 **PC 端 Web 应用** 与 **云端服务** 共同组成。系统以 MQTT 协议为核心通信手段，实现设备数据采集、实时传输、持久化存储、Web 可视化展示及远程指令下发等完整闭环。

系统包含三个独立部署的服务进程：

| 服务 | 端口 | 技术栈 | 面向用户 |
|------|------|--------|----------|
| 后端 API 服务 | 8000 | Python FastAPI + Uvicorn | —（内部服务） |
| 用户前端（User Frontend） | 5176 | Vue 3 + TypeScript + Vite + ECharts + Leaflet | 普通用户 |
| 管理前端（Admin Frontend） | 5175 | Vue 3 + TypeScript + Vite + Element Plus | 系统管理员 |

此外，系统依赖以下基础设施：

| 组件 | 端口 | 说明 |
|------|------|------|
| EMQX Broker | 1883 (MQTT) / 18083 (REST API) | MQTT 消息中间件 |
| MySQL 8.0 | 3306 | 关系型数据库，存储设备/用户/历史数据 |

### 三、数据流全景

系统核心数据流遵循 **发布-订阅** 模式：

```
IoT 设备                    EMQX Broker               FastAPI 后端              MySQL / 前端
───────                    ───────────               ────────────              ────────────

① 上报遥测:
[device_001] ──gps──→  [device/+/gps]  ──订阅──→  MQTTClient._on_message()
                                                     │
                                                     ├─ 未注册? → 丢弃
                                                     │
                                                     ├─ gps → _insert_device_data()
                                                     │          └─ 合并慢遥测缓存 → INSERT INTO device_data
                                                     │
                                                     ├─ telemetry → _merge_slow_telemetry()
                                                     │                └─ 更新缓存 + 回填最新 gps 行
                                                     │
                                                     ├─ alert → _insert_device_data() + _handle_alert()
                                                     │
                                                     ├─ status → _handle_status()
                                                     │            └─ UPDATE device_info SET is_online=
                                                     │
                                                     └─ recording → _handle_recording()
                                                                     └─ INSERT INTO device_recording
                                                                        └─ 关联最近30s告警

② 用户查看:
[浏览器] ──GET /api/device/{id}/latest──→  device_latest()
                                             │
                                             ├─ 校验 user_device_bind (403 if not bound)
                                             ├─ 查询 device_info (在线状态/配置/字段标签)
                                             ├─ 查询最新 telemetry + 最新 gps
                                             ├─ 合并为 latest_raw JSON
                                             └─ 返回 {device_id, is_online, latest_raw, ...}

③ 指令下发:
[浏览器] ──POST /api/device/send_cmd──→  send_device_cmd()
                                           │
                                           ├─ 校验 user_device_bind
                                           └─ mqtt_client.publish_cmd()
                                                └─ MQTT → device/{id}/cmd → [设备执行]
```

### 模块一：MQTT 消息处理模块

**所属文件：** `mqtt_backend/main.py` — `MQTTClient` 类（第 53–424 行）

**功能说明：**
该模块是系统数据入口，以全局单例模式运行于后台线程。通过订阅 EMQX Broker 的 `device/+/#` 通配符主题，接收所有已注册设备的 MQTT 消息，按消息子类型（gps / telemetry / alert / status / recording / config）分发处理，完成数据校验、清洗、合并、持久化全流程。

**设计要点：**
- 全局单例，应用启动时自动连接 EMQX，支持指数退避自动重连（1s→30s）
- 慢遥测数据（telemetry）通过内存缓存 (`_slow_cache`) 暂存，待快遥测（gps）到达时自动合并写入
- 头盔设备（device_001/002/003/helmet）强制电量固定 80%
- 录音（recording）以原始 AMR 二进制存储，并自动关联最近 30 秒内的告警记录
- 未注册设备的消息静默丢弃

| 变量名 | 类型 | 方向 | 说明 |

| `MQTTClient._HELMET_IDS` | `set[str]` | 内部常量 | 头盔设备 ID 集合：`{device_001, device_002, device_003, helmet}` |

### 模块二：用户认证模块

**所属文件：** `mqtt_backend/auth.py`（JWT 签发/校验）、`mqtt_backend/main.py`（路由 488–543 行）、`mqtt_backend/database.py`（密码哈希）

**功能说明：**
实现双通道认证体系：用户端采用 **JWT (HS256, 24h)** Bearer Token 认证；管理端采用 **静态 Admin Token** 放入 `X-Admin-Token` 请求头。密码使用 bcrypt 哈希存储，通过 passlib 库的 `CryptContext` 管理。

### 模块九：数据库模块

**所属文件：** `mqtt_backend/database.py`、`mqtt_backend/schema.sql`、`mqtt_backend/config.py`

**功能说明：**
基于 MySQL 8.0 + DBUtils 连接池实现数据库访问层。使用 `PooledDB` 管理连接复用（min=2, max=5, max_connections=10），配合 `get_conn()` / `get_cursor()` 上下文管理器确保连接正确归还。密码使用 bcrypt 哈希。操作日志写入失败不影响主业务。

**数据库操作核心流程：**

```
╔══════════════════════════════════════════════════════════════╗
║                  数据库连接池 & 操作流程                        ║
╠══════════════════════════════════════════════════════════════╣
║                                                              ║
║   应用启动: check_db_connection()                             ║
║      │                                                       ║
║      ▼                                                       ║
║   ┌──────────────────┐                                       ║
║   │PooledDB 单例初始化│                                       ║
║   │  creator=pymysql  │                                       ║
║   │  mincached=2      │                                       ║
║   │  maxcached=5      │                                       ║
║   │  maxconnections=10│                                       ║
║   │  blocking=True    │  (连接耗尽时等待而非报错)               ║
║   │  cursorclass=     │                                       ║
║   │    DictCursor     │  (查询结果返回字典)                     ║
║   └──────────────────┘                                       ║
║                                                              ║
║   每次请求:                                                   ║
║      │                                                       ║
║      ▼                                                       ║
║   with get_conn() as conn:        # 从池中获取连接             ║
║     with get_cursor(conn) as cur: # 创建 DictCursor           ║
║       cur.execute(SQL, params)    # 参数化查询（防注入）       ║
║       result = cur.fetchall()     # 读取结果                  ║
║     # 自动 cur.close()                                       ║
║     conn.commit()                 # 提交事务                  ║
║   # 自动 conn.close() → 归还连接池                             ║
║                                                              ║
║   异常处理:                                                   ║
║     conn.rollback()  →  不抛出，由路由层 HTTPException 处理   ║
║                                                              ║
║   密码安全:                                                   ║
║     hash_password(plain) → bcrypt 哈希                        ║
║     verify_password(plain, hashed) → bool                     ║
║     使用 passlib CryptContext(schemes=["bcrypt"])             ║
║                                                              ║
║   操作日志:                                                   ║
║     log_operation(user_id, action, target, detail, ip)        ║
║     → INSERT INTO operation_logs                             ║
║     → 失败静默（不影响主业务）                                  ║
║                                                              ║
╚══════════════════════════════════════════════════════════════╝
```

**关键输入/输出变量：**

| 变量名 | 类型 | 方向 | 说明 |
|--------|------|------|------|
| `settings.DB_*` | 多个 | 配置 | 数据库连接参数（host/port/user/pass/db/charset/pool） |
| `_pool` | `PooledDB\|None` | 全局单例 | DBUtils 连接池实例 |
| `conn` | `Connection` | 中间 | 数据库连接，通过 `get_conn()` 上下文管理器获取 |
| `cursor` | `DictCursor/Cursor` | 中间 | 数据库游标，`dict_cursor=True` 返回字典格式 |
| `pwd_context` | `CryptContext` | 工具 | bcrypt 密码加密上下文 |
| `action` | `str` | 日志输入 | 操作类型：login/register_device/delete_device/create_user 等 |
